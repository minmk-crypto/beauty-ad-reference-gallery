# 광고주 관리 시트 가이드

이 갤러리의 **Meta 광고 라이브러리 수집 대상(광고주 목록)**은 마케터가 직접 관리하는 Google Sheet 한 곳에서 정해집니다. 시트를 고치면 다음 갱신 주기(**매주 월·수·금 03:00 KST**)에 자동으로 반영됩니다. 코드를 건드릴 필요가 없습니다.

> TikTok 쪽은 산업·지역 단위 상위 광고 집계라 이 시트와 무관합니다. 이 시트는 **Meta 광고주 추적 목록**만 다룹니다.

---

## 1. 마케터가 하는 일

추적하고 싶은 경쟁사/브랜드를 시트에 한 줄씩 적으면 됩니다. 그게 전부입니다.

- **새 광고주 추가**: 맨 아래에 새 행을 추가합니다.
- **잠시 빼기**: 그 행의 `enabled` 를 `FALSE` 로 바꿉니다(행을 지우지 않아도 됩니다).
- **완전히 제거**: 행을 삭제합니다.

저장만 하면 됩니다(Google Sheet 는 자동 저장). 별도 통보·요청 불필요.

---

## 2. 컬럼 설명

| 컬럼 | 필수 | 설명 | 예시 |
|---|:---:|---|---|
| `label` | ✓ | 갤러리에 표시될 광고주 이름 | `Sephora`, `COSRX` |
| `page_id` | ✓ | Meta 페이지 ID(숫자만). 아래 3번 참고 | `16453004404` |
| `type` | | `retailer`(리테일러) 또는 `brand`(브랜드). 비우면 `brand` | `retailer` |
| `kbeauty` | | K뷰티면 `TRUE`, 아니면 `FALSE`. 비우면 `FALSE` | `TRUE` |
| `country` | | 특정 국가 페이지만 추적할 때 국가코드. 보통은 비웁니다 | `FR` |
| `enabled` | | `FALSE` 면 수집에서 제외. 비우면 수집 | `TRUE` |
| `notes` | | 자유 메모(수집에 영향 없음) | `2026-06 추가` |

표기는 너그럽게 처리됩니다 — `TRUE/FALSE` 대신 `O/X`, `예/아니오`, `리테일러/브랜드`(한글)도 인식합니다. 헤더(첫 행)도 `광고주/페이지id/유형/k뷰티/국가/수집/비고` 같은 한글 이름을 허용합니다. **단, 첫 행은 반드시 헤더여야 하고 `label`·`page_id` 두 컬럼은 있어야 합니다.**

`country` 는 보통 비워둡니다. Meta 글로벌 광고주 페이지는 `country=ALL` 로 수집하기 때문입니다(특정 국가로 보면 "유사 지역 광고" 목록이 떠서 0건이 됩니다). `Sephora France` 처럼 **지역 전용 페이지**를 따로 추적할 때만 국가코드를 적습니다.

---

## 3. Meta 페이지 ID(`page_id`) 찾는 법

1. 추적할 광고주의 [Meta 광고 라이브러리](https://www.facebook.com/ads/library/)를 엽니다.
2. 검색창에 광고주 이름을 넣고 그 광고주를 선택합니다.
3. 주소창 URL 에서 `view_all_page_id=` 뒤의 **숫자**가 `page_id` 입니다.
   - 예: `...&view_all_page_id=16453004404` → `page_id` = `16453004404`

숫자만 적습니다(공백·따옴표는 자동으로 정리되지만, 숫자가 아닌 값이 섞이면 그 행은 건너뛰고 경고로 남습니다).

---

## 4. 언제 반영되나

- **자동**: 매주 **월·수·금 03:00 KST**. 수집이 돌기 직전에 시트를 읽어 추적 목록을 맞춥니다.
- **즉시 확인하고 싶으면**: 저장소 운영자가 GitHub Actions 의 `주간 갱신` 워크플로를 수동 실행(`Run workflow`)할 수 있습니다.

시트가 일시적으로 비어 있거나 접근이 안 되면, **직전 목록을 그대로 유지**합니다(빈 목록으로 도는 사고를 막기 위함). 잘못 적은 행만 건너뛰고 나머지는 정상 반영됩니다.

---

## 5. (운영자용) 최초 1회 셋업

이미 시트가 연결돼 있으면 이 절은 건너뜁니다.

1. **시트 생성**: 새 Google Sheet 를 만들고, 저장소의 [`advertisers-seed.csv`](../advertisers-seed.csv)(현재 추적 중인 광고주 전체)를 가져오기(`파일 → 가져오기 → 업로드`)합니다. 첫 행이 헤더가 되도록 합니다.
2. **마케터 공유**: `공유` 에서 마케터를 **편집자**로 추가합니다.
3. **CSV 게시**(인증 없이 읽기용 — 둘 중 하나):
   - **방법 A — 웹에 게시**: `파일 → 공유 → 웹에 게시 → (해당 시트) + CSV` 선택 → 나오는 URL
     (`https://docs.google.com/spreadsheets/d/e/.../pub?output=csv`).
   - **방법 B — 링크 공유 + gviz**: `공유` 를 `링크가 있는 모든 사용자: 뷰어` 로 두고
     `https://docs.google.com/spreadsheets/d/<시트ID>/gviz/tq?tqx=out:csv&sheet=<탭이름>` 형식 URL 사용.
4. **URL 등록**(둘 중 하나):
   - `config.json` 의 `sources.meta.advertisers_sheet.csv_url` 에 붙여넣고 커밋, **또는**
   - 저장소 `Settings → Secrets and variables → Actions` 에 `ADVERTISERS_SHEET_CSV_URL` 시크릿으로 등록(이 경우 URL 이 커밋되지 않음. 시크릿이 config 값보다 우선).

> 게시되는 데이터는 공개 경쟁사 페이지 ID 목록이라 민감정보가 아닙니다. 방법 A 가 CI 에서 가장 안정적입니다.

### 동작 점검

```bash
# 로컬에서 한 번 돌려 동기화 결과(추가/제거 요약)를 확인
ADVERTISERS_SHEET_CSV_URL="<게시 CSV URL>" node sync-advertisers.js
# config.sources.meta.advertisers 에 시트 내용이 반영됩니다.
```
